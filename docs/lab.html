Core Results

基準組最重要的五件事

Filesystem policy 成立

`/tmp` 可寫、`/var/tmp` 被拒絕、根目錄列舉被拒。這是 OpenShell 在 runc 路徑真正成立的代表性能力。

L7 控制完整

同一個 endpoint，`curl GET` 可以，`python3 GET` 不行，`curl POST` 也不行。這證明 binary 與 method/path 都有被判斷。

Policy 可熱重載

Pod UID 不變，代表新政策不是靠重建 Pod 才生效。這是 OpenShell control plane 的核心價值之一。

FUSE baseline 成立

普通 privileged Pod 已可在這條路徑上用 `sshfs` 掛遠端目錄，說明底層 k3s + runc 本身沒有擋住 FUSE。

Azure File CSI 成立

同一顆 RWX PVC 已成功被普通 Pod 和 Sandbox 共享讀寫，證明 CSI-backed filesystem 可以直接進入 Sandbox。

限制也已釐清

OpenShell-managed sandbox 若要寫入新掛載路徑，還是要把路徑加入 `filesystem_policy.read_write`，並重建 sandbox。
