CORE RESULTS

Why this line remains the baseline

Filesystem policy holds

`/tmp` stays writable, `/var/tmp` is denied, and root directory listing is blocked. This is the strongest proof that Landlock-backed enforcement is active on runc.

L7 policy stays intact

The same endpoint allows `curl GET`, denies `curl POST`, and denies `python3 GET`, showing method/path and binary controls both remain in play.

Hot-reload is real

Policy updates apply without forcing a pod rebuild. Matching Pod UIDs before and after the update are the key evidence point.

FUSE baseline works

A standard privileged pod can mount `sshfs` successfully here, which proves the underlying k3s + runc path is not what blocks later sandbox-specific FUSE experiments.

Azure File CSI works in Sandbox

The same RWX PVC is shared successfully between a standard pod and an OpenShell sandbox, which establishes the CSI-backed storage path on the baseline runtime.

Limits are explicit

New writable mount paths still need to be declared in `filesystem_policy.read_write`, and the sandbox must be recreated for the updated mount policy to apply.
