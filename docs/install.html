Install

Install the Azure Kata environment

The verified installation path uses a dedicated Terraform stack plus a wrapper script that provisions Azure, installs K3s with CRI-O, installs Kata on every node, and verifies the runtime.

Required Files

Files used by the install path

These are the exact repo entry points for the Kata Azure environment.

  • Stack: terraform/stacks/k3s-kata-134/main.tf
  • Stack variables: terraform/stacks/k3s-kata-134/variables.tf
  • Stack example values: terraform/stacks/k3s-kata-134/stack.auto.tfvars.example
  • One-shot install wrapper: scripts/install-k3s-kata-134.sh
  • Prerequisite check: scripts/check-kata-prereqs.sh
  • Kata install logic: scripts/install-kata.sh

One-Shot

Run the full install

The wrapper uses the verified defaults for K3s and CRI-O unless you override them.

bash scripts/install-k3s-kata-134.sh
  • K3s default: v1.34.1+k3s1
  • CRI-O stream default: v1.34
  • VM size in the stack: Standard_D4s_v3
  • Kata release default in the install script: 3.31.0
What It Does

Wrapper sequence

The install wrapper performs all required stages in a fixed order.

  1. Apply the Azure stack in terraform/stacks/k3s-kata-134.
  2. Fetch the kubeconfig into generated/stacks/k3s-kata-134/kubeconfig.
  3. Wait for all 3 nodes to become Ready.
  4. Run scripts/check-kata-prereqs.sh to confirm hardware virtualization and /dev/kvm.
  5. Run scripts/install-kata.sh on the control plane and both workers.
  6. Apply k8s/kata-runtimeclass.yaml.
  7. Run scripts/verify-kata-runtime.sh to launch a Kata-backed probe pod.
Manual Steps

Run the install in smaller pieces

Use these commands if you want to re-run only part of the flow.

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
TF_DIR=terraform/stacks/k3s-kata-134 \
bash scripts/check-kata-prereqs.sh

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
TF_DIR=terraform/stacks/k3s-kata-134 \
bash scripts/install-kata.sh

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
bash scripts/verify-kata-runtime.sh
CRI-O Wiring

What Kata changes on each node

The install script writes two CRI-O drop-ins before restarting services.

[crio]
storage_option = [
  "overlay.skip_mount_home=true",
]

[crio.runtime.runtimes.kata]
runtime_path = "/usr/local/bin/containerd-shim-kata-v2"
runtime_type = "vm"
runtime_root = "/run/vc"
privileged_without_host_devices = true
