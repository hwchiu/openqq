Raw artifacts from the latest Kata run

Artifact List

Stored evidence files

Each file captures one part of the install or verification story.

  • versions.txt Kubernetes client and server versions plus kubectl cluster-info.
  • nodes-wide.txt Node readiness, OS image, kernel, and CRI-O runtime version.
  • prereqs.txt The latest stored output of the Kata prerequisite check, including /dev/kvm.
  • crio-kata-dropins.txt The CRI-O storage and Kata runtime drop-ins captured from all nodes.
  • runtimeclass-kata.yaml The applied RuntimeClass object.
  • kata-verify-pod.yaml The verification pod manifest used by the script.
  • kata-evidence-pod.yaml The completed pod as captured from the cluster.
  • kata-evidence-logs.txt The pod log with guest kernel details and kata-probe-ok.
  • pods-wide.txt Pod placement details from the verification window.
  • events.txt Kubernetes event stream captured during the verification window.
