Evidence

Raw artifacts from the latest Kata run

The raw evidence for this environment is split into two stored snapshots: records/raw/2026-06-25/k3s-kata-134-runtime-verify/ for the original runtime verify and records/raw/2026-06-28/kata-proof-3-live-inspect/ for a stronger live proof chain. Because GitHub Pages cannot serve those paths directly, the links below open the corresponding GitHub files.

Artifact List

Stored evidence files

Each file captures one part of the install or verification story.

  • versions.txt Kubernetes client and server versions plus kubectl cluster-info.
  • nodes-wide.txt Node readiness, OS image, kernel, and CRI-O runtime version.
  • prereqs.txt The latest stored output of the Kata prerequisite check, including /dev/kvm.
  • crio-kata-dropins.txt The CRI-O storage and Kata runtime drop-ins captured from all nodes.
  • runtimeclass-kata.yaml The applied RuntimeClass object.
  • kata-verify-pod.yaml The verification pod manifest used by the script.
  • kata-evidence-pod.yaml The completed pod as captured from the cluster.
  • kata-evidence-logs.txt The pod log with guest kernel details and kata-probe-ok.
  • pods-wide.txt Pod placement details from the verification window.
  • events.txt Kubernetes event stream captured during the verification window.
2026-06-28 Live Inspect

Node-side proof that the pod was using Kata

These artifacts were captured from a long-running live proof pod and the worker node that hosted it.

  • pod-wide.txt Live pod placement showing kata-proof-3 running on worker-2.
  • pod.yaml Live pod spec with runtimeClassName: kata.
  • pod-logs.txt Guest-kernel output from inside the running Kata pod.
  • crictl-inspectp.json CRI-O sandbox inspection showing io.kubernetes.cri-o.RuntimeHandler = "kata".
  • crictl-inspect.json CRI-O container inspection tied to the same sandbox ID.
  • kata-processes.txt Worker-node process list showing containerd-shim-kata-v2, qemu-system-x86_64, and virtiofsd.
  • crio-journal.txt CRI-O journal lines for sandbox creation and container start on the Kata proof pod.
  • sandbox-id.txt The sandbox ID used to correlate Kubernetes, CRI-O, shim, and qemu evidence.
