原始證據

這次 Kata 驗證保存了哪些 artifacts

目前的 raw evidence 分成三批：`2026-06-25` 保存初次 runtime verify，`2026-06-28` 保存 node-side live inspect，另外同一天也保存了一次 `Istio + Kata` smoke。由於 GitHub Pages 不會直接提供 `records/raw` 路徑，所以以下連結會直接打到 GitHub blob。

2026-06-25 Runtime Verify

初次 runtime verify 保存的檔案

這一批證據主要證明叢集成功安裝 Kata，並且基本 verify pod 可完成執行。

  • versions.txt Kubernetes client/server 版本與 kubectl cluster-info。
  • nodes-wide.txt 節點 readiness、OS image、kernel 與 CRI-O runtime 版本。
  • prereqs.txt Kata prerequisite check，包括 /dev/kvm。
  • crio-kata-dropins.txt 每個節點上的 CRI-O storage 與 Kata runtime drop-ins。
  • runtimeclass-kata.yaml 已套用的 RuntimeClass 物件。
  • kata-verify-pod.yaml verify script 使用的 pod manifest。
  • kata-evidence-pod.yaml 已完成的 pod 狀態與 spec。
  • kata-evidence-logs.txt pod 內的 guest kernel 與 kata-probe-ok。
  • pods-wide.txt 驗證視窗內的 pod placement 資訊。
  • events.txt 驗證期間的 Kubernetes event stream。
2026-06-28 Live Inspect

證明 pod 真正在使用 Kata 的節點側證據

這一批證據來自一個長時間存活的 proof pod，以及承載它的 worker node。

  • pod-wide.txt 顯示 kata-proof-3 實際跑在 worker-2。
  • pod.yaml live pod spec，內含 runtimeClassName: kata。
  • pod-logs.txt 從 pod 內看到的 guest kernel 輸出。
  • crictl-inspectp.json sandbox inspect 證據，內含 io.kubernetes.cri-o.RuntimeHandler = "kata"。
  • crictl-inspect.json container inspect 證據，可對應同一個 sandbox ID。
  • kata-processes.txt worker node 上的 containerd-shim-kata-v2、qemu-system-x86_64 與 virtiofsd。
  • crio-journal.txt crio journal 中 proof pod 的 sandbox 與 container 啟動記錄。
  • sandbox-id.txt 用來串起 Kubernetes、CRI-O、shim 與 qemu 證據的 sandbox ID。
  • node-name.txt 這次 proof pod 落在哪個節點上。
  • node-host.txt 承載 proof pod 的 worker node 對外 IP。
2026-06-28 Istio + Kata

Kata sidecar workload 的 mesh 驗證證據

這一批證據證明 Istio sidecar 疊加到 workload 之後，pod 仍然維持 Kata runtime。

  • istio-chart-version.txt 這次 live 安裝採用的 Istio chart 版本字串。
  • istio-install.log 安裝 istio-base 與 istiod 時留下的完整 transcript。
  • istio-control-plane.json istiod control plane Ready 結果。
  • helm-istio-system.json Istio system namespace 內的 Helm release 狀態。
  • istio-system-pods.txt `istio-system` 內實際運行中的 pod。
  • istio-kata-sidecar.json smoke 測試結果，包含 namespace、server/client pod 名稱、node 與回應內容。
  • pods-wide.txt smoke namespace 內的 pod placement 與狀態。
  • server-pod.yaml echo pod 的完整 spec，內含 runtimeClassName: kata 與 sidecar.istio.io/status。
  • client-pod.yaml curl pod 的完整 spec，內含 runtimeClassName: kata 與 sidecar.istio.io/status。
  • server-crictl-inspectp.json server sandbox 的 CRI-O inspect 證據，內含 RuntimeHandler = "kata"。
  • client-crictl-inspectp.json client sandbox 的 CRI-O inspect 證據，內含 RuntimeHandler = "kata"。
  • kata-processes.txt worker node 上兩個 sandbox 對應的 containerd-shim-kata-v2、qemu-system-x86_64 與 virtiofsd。
  • crio-journal.txt `crio` journal 裡 server / client pod 的 sandbox 與 sidecar container 啟動紀錄。
