這輪最重要的修正

  • 四套叢集統一升級到 `K3s 1.31 + CRI-O 1.31`。
  • 修掉 CRI-O 路徑上的 `bandwidth` CNI plugin panic，恢復 workload 建立。
  • `gVisor` 路線補上 `drop_infra_ctr=false`、`/run/runsc` 與 `runsc` runtime handler。
  • Istio smoke test 改成每次使用獨立 namespace，避免 `Terminating` namespace 污染 rerun。
  • `gvisor-runtime` runner 改成失敗快速返回，避免整輪 matrix 卡死。

該怎麼讀這份結果

  • `OpenShell + runc` 仍是最穩的完整主線。
  • `OpenShell + gVisor` 這輪不要再簡化成「一定 degraded」；Guardrails 是 PASS，但 bare runtime probe 與 gVisor+sidecar 仍 FAIL。
  • `KubeArmor + runc` 不能只看 token block；secret/file 有效果，但 process/network 仍沒有擋住。
  • 一般 Istio sidecar injection 不等於 `RuntimeClass gvisor` + sidecar 可用，這兩件事在這輪被明確分開了。
