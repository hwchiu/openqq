EXECUTIVE REPORT Decision summary

Platform decision,
without the noise

The shortest defensible conclusion is still that `OpenShell + runc` remains the only path preserving the full OpenShell control surface. gVisor is the deliberate runtime-isolation trade-off. Kata remains blocked for the OpenShell sandbox path in this lab.

See the runc baseline Open the matrix
Shortest verdict

Use runc + OpenShell when feature completeness matters. Use gVisor only when the stronger runtime boundary is worth losing filesystem policy. Treat Kata as blocker analysis, not production guidance.

Recommended

runc + OpenShell

This is still the most complete line: filesystem policy, network control, L7 filters, binary allowlist, hot-reload, and the downstream storage experiments all remain intact.

Trade-off

gVisor + OpenShell

This line buys a stronger runtime boundary, but it does not preserve Landlock-backed filesystem policy. L7 and hot-reload still work, which keeps it relevant as a deliberate compromise path.

Blocked

Kata + OpenShell

The blocker is no longer ambiguous: the OpenShell sandbox path still requires a privileged workload shape that the current Kata path rejects in this environment.
