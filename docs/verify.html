驗證

如何確認 Kata 已經真的啟用

這裡的驗證分成兩層：第一層是 host prerequisite，第二層是 Kubernetes 與節點側的 runtime 行為。兩層都需要，才能把「有設設定」和「真的跑在 Kata VM sandbox 裡」區分開來。

快速指令

如何重跑目前的驗證

以下指令與 repo 中目前保存的驗證流程一致。

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
TF_DIR=terraform/stacks/k3s-kata-134 \
bash scripts/check-kata-prereqs.sh

kubectl --kubeconfig generated/stacks/k3s-kata-134/kubeconfig get nodes -o wide
kubectl --kubeconfig generated/stacks/k3s-kata-134/kubeconfig get runtimeclass kata -o yaml

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
bash scripts/verify-kata-runtime.sh

PATH="$HOME/.local/bin:$PATH" \
scripts/install-istio-stack.sh k3s-kata-134 generated/stacks/k3s-kata-134/kubeconfig

KEEP_NAMESPACE=true \
scripts/tests/istio-kata-sidecar.sh generated/stacks/k3s-kata-134/kubeconfig k3s-kata-134

KEEP_NAMESPACE=true \
scripts/tests/networkpolicy-kata-ingress.sh generated/stacks/k3s-kata-134/kubeconfig k3s-kata-134
