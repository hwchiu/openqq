Verify

Verify that Kata is active

Verification has two layers: host prerequisites and Kubernetes runtime behavior. Both were run on the latest Azure environment.

Quick Commands

Re-run the latest checks

These commands match the scripts used during the latest verification run.

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
TF_DIR=terraform/stacks/k3s-kata-134 \
bash scripts/check-kata-prereqs.sh

kubectl --kubeconfig generated/stacks/k3s-kata-134/kubeconfig get nodes -o wide
kubectl --kubeconfig generated/stacks/k3s-kata-134/kubeconfig get runtimeclass kata -o yaml

KUBECONFIG_PATH=generated/stacks/k3s-kata-134/kubeconfig \
bash scripts/verify-kata-runtime.sh
