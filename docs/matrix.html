四套環境
用同一份測試語言比較

這頁不再讓每個 runtime 各寫各的故事。它把四套安裝路線拉回同一個 matrix：同一批 test case、同一批 status、同一批證據入口。這才是之後能持續維護的技術文件骨架。

看 matrix runbook 看 test catalog

標準流程

./scripts/install-comparison-matrix.sh
PUBLISH_RESULTS=true ./scripts/run-comparison-matrix-tests.sh

狀態語言

  • PASS: 測試成立
  • DEGRADED: 功能存在，但有已知退化
  • PENDING: 腳本已準備，但還沒發佈最新實測
  • N/A: 該環境不適用
